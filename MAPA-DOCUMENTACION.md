## Criterio de prioridad documental

Cuando haya diferencias entre documentos, la prioridad recomendada es esta:

1. `Documentacion/informeIA.md` para visión futura.
2. `ESTADO-ACTUAL.md` para estado real del repositorio.
3. Documentación interna de `normativa-agent/` para operación del MVP.
4. Documentos históricos de prueba o experimentación.
